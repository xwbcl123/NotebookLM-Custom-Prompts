# [EU-DRULE-ADDON-03] 报告格式模板库 – 欧盟数字政策写作 Prompt 工具包 v1.0

> 角色定位（给 Agent）：  
> - 你正在访问的是一个「报告格式模板库」，用于支持 EU-DRULE 系统下的自定义报告生成。  
> - 当上层 ADDON（例如 ADDON-02 `/rprompt`）需要为某个主题生成自定义报告 Prompt 时，应先在本文件中寻找**最接近的模板**，理解其结构与参数，再结合当前主题进行改写。  
> - 不要直接生搬硬套模板文字，而是将其视为 few-shot 示范：学习**结构、参数含义、风格要点**，并据此产生新的 Prompt。

---

## 🧭 使用说明（给人类用户）

本工具包整理了 10 类常用的欧盟数字政策/法规写作场景，均以「可复用、可填空的 Prompt 模板」形式呈现，方便在 ChatGPT、Gemini、NotebookLM 等环境中一键调用。

- 每个模板都包含三部分：
  - **📌 模板正文**：可直接复制，替换中括号 `[参数]` 即可使用；
  - **📊 参数填空表**：帮助你快速理解、填写各个参数；
  - **✨ 输出风格要点**：隐含的写作风格与结构要求。
- 建议用法：
  - 先在你的工具中粘贴模板；
  - 按实际项目场景替换参数；
  - 如需多语种版本，可在模板后追加语言要求（如 “请用中英双语输出”）。

> 提示：在 EU-DRULE 体系内，本文件通常由上层的 `/rprompt`、`/briefing-prompt` 等命令间接调用，人类用户也可以直接复制对应模板用于一次性写作。

## 1. 影响分析报告

### 1.1 模板正文

> 关于 **[Theme]** 对 **[Entity]** 的影响和机遇的分析报告。

根据源文件信息，撰写一份关于拟议的 **[Theme]** 对 **[Entity]** 的潜在影响的综合分析。报告应以专业的口吻，详细阐述该 **[Theme]** 如何解决 **[Entity]** 面临的特定 **[Challenge-Type, 如 行政与合规挑战]**，评估预期的 **[Benefit-1, 如 成本节约]** 和 **[Benefit-2, 如 效率提升]**，并探讨可能出现的 **[Opportunity-Type, 如 新的市场机会 / 商业模式]**。本分析旨在为 **[Stakeholders, 如 政策制定者 / 行业协会 / 企业决策者]** 提供决策参考。

### 1.2 参数填空表

| 参数名 | 示例 | 说明 |
|--------|------|------|
| [Theme] | 新数字法规包、绿色转型计划 | 被评估的主题或举措 |
| [Entity] | 欧洲中小企业、电信运营商、平台企业 | 主要受影响对象 |
| [Challenge-Type] | 行政负担、合规成本、跨境运营复杂性 | 该群体当前面临的痛点 |
| [Benefit-1/2] | 监管成本下降、流程自动化 | 可量化或定性的收益 |
| [Opportunity-Type] | 新市场、数据驱动服务、合作生态 | 由改革带来的新机遇 |
| [Stakeholders] | 政府部门、行业协会、企业管理层 | 报告的目标读者 |

### 1.3 输出风格要点

- 结构建议：**背景 → 现状挑战 → 改革机制 → 影响分析 → 机遇与风险 → 建议**；
- 明确区分**事实（Fact）**与**判断（Judgement）**；
- 可使用表格、分点列出不同类型主体的影响差异；
- 适合用作高层简报或内部研究报告基础。

## 2. 白皮书

### 2.1 模板正文

> 关于 **[Theme]** 对 **[Entity]** 影响的综合白皮书。

撰写一份详细的白皮书，阐述 **[Policy / Initiative / Program]** 对 **[Entity]** 的长期与系统性影响。文稿应以专业、客观的口吻，深入探讨 **[Theme]** 中的核心机制（如 **[Mechanism-1]**、**[Mechanism-2]**、**[Mechanism-3]**）如何重塑 **[Industry / Domain]** 的监管环境、降低行政成本，并为创新与市场增长创造机会。重点分析对 **[Entity]** 的具体影响，包括采用成本、合规挑战和潜在竞争优势。本白皮书旨在为 **[Stakeholders, 如 政策制定者 / 行业协会 / 企业领袖]** 提供中长期战略参考。

### 2.2 参数填空表

| 参数名 | 示例 | 说明 |
|--------|------|------|
| [Theme] | 欧盟数字一揽子计划、AI 监管新框架 | 白皮书聚焦的整体主题 |
| [Policy / Initiative / Program] | 《数字综合法案》、数据法案、AI 法案 | 具体政策或计划 |
| [Entity] | 中小企业、金融机构、公共部门 | 被分析的主体 |
| [Mechanism-1/2/3] | 数据共享机制、统一身份体系、事件报告制度 | 具有系统性影响的关键机制 |
| [Industry / Domain] | 数字经济、工业制造、医疗健康 | 所属行业或领域 |
| [Stakeholders] | 政策制定者、标准化机构、龙头企业 | 主要参考、使用人群 |

### 2.3 输出风格要点

- 适合采用「白皮书」常见结构：**前言 → 背景 → 问题定义 → 政策/技术框架 → 影响分析 → 路线图/建议**；
- 叙述风格偏专业、稳重，可适度加入图示（框架图、时间轴、生态图）；
- 强调中长期趋势与战略意义，而非只做短期解读。

## 3. 战略分析报告

### 3.1 模板正文

> 评估 **[Regulatory-Theme]** 带来的机遇和合规风险的战略分析报告。

准备一份面向 **[Primary-Audience-1]** 和 **[Primary-Audience-2]** 的战略分析报告，评估 **[Regulatory-Package]** 带来的机遇与挑战。报告应重点分析 **[Instrument-A]** 的实施对 **[Business-Dimension-1]**、**[Business-Dimension-2]** 和 **[Business-Dimension-3]** 的潜在影响。  

同时，深入探讨 **[Instrument-B]** 对 **[Regulatory-Domain-1]**、**[Regulatory-Domain-2]** 和 **[Regulatory-Domain-3]** 相关法规的 **[Change-Pattern, 如 简化 / 强化 / 统一]** 所带来的合规调整与成本效益。报告需采用严谨的分析方法，为 **[Target-Entity]** 制定应对策略提供依据。

### 3.2 参数填空表

| 参数名 | 示例 | 说明 |
|--------|------|------|
| [Regulatory-Theme] | 欧盟数字法规变化、跨境数据流动新规 | 报告总体评估主题 |
| [Regulatory-Package] | Digital Package、AI 监管组合拳 | 一揽子/组合政策名称 |
| [Instrument-A/B] | 欧洲商业钱包、数字综合法案、数据法案 | 具体关键工具或法律文件 |
| [Business-Dimension-1/2/3] | 运营效率、数据管理流程、跨境交易 | 受影响的业务维度 |
| [Regulatory-Domain-1/2/3] | 数据保护、网络安全、AI 治理、竞争法 | 涉及的法规领域 |
| [Change-Pattern] | 简化合规、统一门槛、提高监管强度 | 法规变化的主要方向 |
| [Primary-Audience-1/2] | 企业高管、合规官、法务负责人 | 报告核心读者 |
| [Target-Entity] | 在欧运营企业、全球供应链企业、中小企业 | 需要据此制定策略的对象 |

### 3.3 输出风格要点

- 重点是「**战略含义**」而非条文逐条解读；
- 建议加入「情景分析」「风险-机会矩阵」等工具；
- 适合作为管理层决策简报或内部战略讨论输入。

## 4. 概念解析

### 4.1 模板正文

> 用简单语言解释 **[Policy / Strategy]** 及其核心组成部分。

为 **[Audience-Segment, 如 初学者 / 普通民众 / 学生群体]** 撰写一份清晰易懂的入门介绍。本文档应以通俗化表达解释 **[Policy / Strategy]** 的总体目标，并分别阐述其 **[Pillar-Count]** 个核心组成部分（例如：**[Pillar-1]**、**[Pillar-2]**、**[Pillar-3]** 等）的作用与意义。  

目标是帮助读者建立对 **[Domain / Sector]** 转型方向与战略布局的基本理解，并认识这些措施如何影响 **[Relevant-Impact-Area]**（如商业活动简化、创新促进、消费者权益保障等）。

### 4.2 参数填空表

| 参数名 | 示例 | 说明 |
|--------|------|------|
| [Policy / Strategy] | 数字一揽子计划、AI 国家战略、数据联盟战略 | 需要解释的战略/政策 |
| [Audience-Segment] | 普通公众、跨国企业新人、大学生 | 目标读者 |
| [Pillar-Count] | 三大支柱、四个核心方向 | 组成要素个数描述 |
| [Pillar-1/2/3] | 数字综合法案、数据联盟战略、欧洲商业钱包 | 核心组成要素 |
| [Domain / Sector] | 数字化转型、医疗、教育、交通 | 所属领域 |
| [Relevant-Impact-Area] | 商业活动、跨境服务、科研合作、隐私保护 | 希望读者理解的影响维度 |

### 4.3 输出风格要点

- 面向初学者，避免使用未解释的专业术语；
- 适合用类比、场景化例子（如“就好像为企业建立一个统一的数字身份证……”）；
- 结构上建议：**是什么 → 有哪些部分 → 各部分做什么 → 整体有什么用**。

## 5. 解释性文章

> 本节包含两种常见用法：**工具/机制类解释** 和 **战略/政策类解释**。可按需要选用其一。

### 5.1 工具/机制类解释性文章

#### 5.1.1 模板正文

> 解释 **[Concept / Tool]** 将如何简化 **[Target-Activity / Business-Process]**。

撰写一篇解释性文章，帮助 **[Audience-Type, 如 学生 / 商业新人 / 普通用户]** 理解 **[Concept / Tool]** 的核心概念及其带来的实际好处。文章应使用贴近生活的例子，说明该工具如何改善 **[Target-Activity / Business-Process]**（如减少文书工作、提升流程效率、简化身份验证与数据交换），从而节省 **[Resource-Type, 如 时间 / 成本 / 人力]**。文章应以 **[Tone, 如 积极 / 鼓励 / 普及]** 的口吻呈现，展现数字化手段带来的便利与价值。

#### 5.1.2 参数填空表

| 参数名 | 示例 | 说明 |
|--------|------|------|
| [Concept / Tool] | 欧洲商业钱包、电子发票平台、在线签名系统 | 被解释的数字工具或机制 |
| [Target-Activity / Business-Process] | 企业行政管理、跨境贸易文件提交、合同签署流程 | 工具改进的业务场景 |
| [Audience-Type] | 商学院学生、小微企业主、新入职员工 | 目标读者 |
| [Resource-Type] | 时间、资金、人力成本 | 主要节省的资源 |
| [Tone] | 鼓励、积极、友好、易懂 | 写作语气要求 |

#### 5.1.3 输出风格要点

- 使用小故事或场景对话来说明前后差异；
- 少量技术细节 + 大量“这对你意味着什么”的解释；
- 适合发布在科普文章、企业博客或入门读物中。

---

### 5.2 战略/政策类解释性文章

#### 5.2.1 模板正文

> 一篇为初学者解释 **[Strategy / Policy]** 及其 **[Key-Objectives-Count]** 项核心目标的文章。

撰写一篇清晰易懂的文章，面向 **[Audience Type, 如 学生 / 普通公众 / 企业新人]** 解释什么是 **[Strategy / Policy]**。文章应以简洁的语言，重点阐述该战略旨在应对的 **[Key Challenge-1]**、**[Key Challenge-2]**、**[Key Challenge-3]** 等核心挑战，并说明其对 **[Outcome Area, 如 技术发展 / 市场竞争 / 公共服务]** 的意义。

#### 5.2.2 参数填空表

| 参数名 | 示例 | 描述 |
|--------|------|------|
| [Strategy / Policy] | 数据联盟战略、网络韧性战略、AI 促进计划 | 要解释的战略或政策 |
| [Audience Type] | 学生、公众、企业新人 | 目标读者 |
| [Key-Objectives-Count] | 三大目标、四项重点任务 | 核心目标数量 |
| [Key Challenge-1/2/3] | 数据稀缺、法规复杂性、全球竞争压力 | 战略试图解决的关键问题 |
| [Outcome Area] | AI 发展、国际竞争力、产业增长、社会福祉 | 战略意义的主要落点 |

#### 5.2.3 输出风格要点

- 重点在“**为什么需要这项战略**”和“**它打算怎么做**”；
- 用日常语言，避免政策文件式的生硬表达；
- 适合用于课堂讲解材料、政策入门手册等。

## 6. 立法分析

### 6.1 模板正文

> 一份全面的分析报告，旨在解读 **[Jurisdiction]** 的 **[Legislative-Framework]** 最新修订。

根据提供的 **[Source-of-Law / Proposal Type]**，生成一份详细的立法分析。该分析应阐明拟议的 **[Law / Act / Directive Name]** 的 **[Objectives]**、**[Scope]** 和 **[Key-Provisions]**。  

专业地解释其对 **[Regulatory-Domain-1]**、**[Regulatory-Domain-2]**、**[Regulatory-Domain-3]**、**[Regulatory-Domain-4]** 的潜在影响。本文件应保持客观与信息丰富的语气，旨在帮助 **[Audience Type, 如 专业人士 / 合规官 / 政策研究者]** 理解新的监管环境并评估合规要求。

### 6.2 参数填空表

| 参数名 | 示例 | 注释说明 |
|--------|------|----------|
| [Jurisdiction] | 欧盟、英国、美国、东盟 | 立法适用地区 |
| [Legislative-Framework] | 数字法规体系、数据治理框架、AI 监管体系 | 所属法规框架 |
| [Source-of-Law / Proposal Type] | 欧委会提案、条例草案、监管指南 | 分析依据来源 |
| [Law / Act / Directive Name] | 数字综合法案、数据法案、AI 法案 | 具体法律名称 |
| [Objectives] | 风险管控、统一市场、促进创新 | 法规目标 |
| [Scope] | 适用对象、涵盖领域、地域范围 | 法规适用范围 |
| [Key-Provisions] | 关键义务、豁免条款、执行机制 | 法规核心条款 |
| [Regulatory-Domain-1/2/3/4] | 数据保护、网络安全、AI 治理、竞争法 | 涉及监管领域 |
| [Audience Type] | 合规官、政策分析师、行业协会、企业法务 | 报告读者群体 |

### 6.3 输出风格要点

- 法律与政策分析风格，语言精准、克制；
- 突出 “**有什么新变化**”“**与既有法规的关系**”；
- 可设置对比表（新旧条款对比）、时间线（立法进程）、影响分类。

## 7. 影响评估报告

### 7.1 模板正文

> 一份详细报告，评估 **[Proposed-Law / Policy]** 对 **[Target-Entity]** 运营和合规策略的影响。

利用源文件中的信息，撰写一份详细的影响评估报告。报告应重点分析 **[Proposed-Law / Policy]** 对 **[Entity-Group-1]**、**[Entity-Group-2]** 等不同规模或类型组织的实际影响。  

内容应覆盖：  
- **[Regulatory-Change-1]**  
- **[Regulatory-Change-2]**  
- **[Regulatory-Change-3]**  
- 对现行法规（如 **[Existing-Reg-X]**、**[Existing-Reg-Y]**）的衔接与修订  

并评估这些变化可能带来的：  
- **[Cost/Benefit Type-1, 如 成本节约]**  
- **[Cost/Benefit Type-2, 如 管理负担变化]**  
- **[Competitive-Angle, 如 竞争优势或市场准入]**  

该报告应为企业提供战略洞见，帮助其驾驭不断变化的监管环境。

### 7.2 参数填空表

| 参数名 | 示例 | 说明 |
|--------|------|------|
| [Proposed-Law / Policy] | 数字综合法案、跨境数据流动框架、AI 监管更新 | 拟议中的法规或政策 |
| [Target-Entity] | 企业运营与合规团队、金融机构、供应链企业 | 总体受影响主体 |
| [Entity-Group-1/2] | 中小企业 / 中型企业、平台公司 / 制造企业 | 分组分析的对象 |
| [Regulatory-Change-1/2/3] | 事件报告要求变更、数据共享制度优化、统一身份认证 | 关键变更点 |
| [Existing-Reg-X/Y] | GDPR、NIS2、AI Act、eIDAS | 既有重要法规 |
| [Cost/Benefit Type-1/2] | 合规成本下降、流程精简、IT 投入变化 | 成本/收益维度 |
| [Competitive-Angle] | 创新激励、市场准入门槛变化、品牌信任度提升 | 战略与市场层面的影响 |

### 7.3 输出风格要点

- 建议采用「主体类别 × 影响维度」的矩阵结构；
- 在结尾部分明确给出「短期应对」「中期调整」「长期布局」等层次化建议；
- 适合作为企业内部合规与战略部门联合阅读材料。

## 8. 法规摘要

### 8.1 模板正文
