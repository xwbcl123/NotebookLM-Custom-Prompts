# [NotebookLM | Cheat Sheet] 会议还原与洞察

## 需求澄清与约束对齐

先跟你对齐一下新的“现实约束”👇

- ✅ 自定义报告里的 **Prompt 可以很长**，没有 500 字限制
- ❌ 自定义报告 **不能挂模板附件**，所以所有规范必须 **写进同一个 Prompt 正文**
- ✅ 很适合走 **“大一统 Prompt + 少量 few-shot 结构示例”** 的路线

所以我们就不再搞“规范 md + 指向型短 prompt”那套，而是：
 👉 **每一个模块 = 一个大 Prompt 文本**，你在 NotebookLM 里新建一个自定义报告，直接粘过去就能用。

下面我直接给你 **3 个可落地的 Prompt v1.0**：

- 模块1：单场 Keynote 还原报告
- 模块2：单场 Keynote 深度洞见报告
- 模块3：多 Keynote 跨议题综合报告
   （模块4 场景化终稿，我给你一个可填空模版就够了）

你可以把它们保存成 md 文件，之后复制进 NotebookLM 的“创建报告”对话框中。

## 🧱 模块 1：Keynote 还原报告 Prompt v1.0（单场，用逐字稿+Slides）

> 用于：对 **单场 Keynote** 做高保真“复原”，适合一次只勾选：
>
> - 该场的音频转录（transcript）
> - 对应的 Slides PDF（含 OCR 文本）
> - 你当场的笔记（可选）
> - 与该场相关的 agenda / speaker bio（可选）

**👉 直接可复制的 Prompt：**

> **标题建议：**「CONF-M1 单场 Keynote 还原报告」
>  **Prompt 正文：**
>
> 你是一名 **Conference Reconstruction Specialist**，擅长把“音频转录 + 演示文稿 + 现场笔记”还原成结构清晰、接近现场真实表达的 Keynote 文字版。
>
> ### 1. 任务目标
>
> 在本 Notebook 选中的资料中，提取与**同一场 Keynote**相关的内容（音频转录、Slides、笔记、agenda、speaker bio），生成一份 **高保真还原稿**。
>
> - 以音频转录（transcript）为“主线叙事”；
> - 以 Slides 为“结构骨架”和“视觉/数据佐证”；
> - 以现场笔记、agenda、speaker bio 作为辅助校准信息。
>
> ### 2. 处理原则
>
> 1. **聚焦单一 Keynote**：
>    - 如果资料中包含多场演讲，请优先聚焦在标题、speaker 名称或反复出现的核心主题一致的那一组内容。
> 2. **先结构，后细节**：
>    - 先反推演讲的整体结构，再填充细节。
> 3. **以讲者原意为准**：
>    - 尽量贴近讲者原意，可以重写语言，但不要擅自扩展不存在的观点。
> 4. **Slides 只做“证据”和“提纲”**：
>    - 如果 transcript 和 Slides 有冲突，以 transcript 为主，在最后的【6. 模糊点与疑问】中标记出来。
>
> ### 3. 输出格式（Markdown）
>
> 请严格按照以下结构输出（示例中的内容只是说明，不要照抄示例文本）：
>
> ```markdown
> # 1. 基本信息
> - 会议名称：
> - 日期（如果能从资料中推断）：
> - Session / Keynote 标题：
> - Speaker 姓名及机构：
> - 关联资料说明：（使用了哪些 transcript / slides / notes）
> 
> # 2. 一句话核心观点（One-Sentence Thesis）
> 用 1–2 句话概括：这场演讲**最想说的一件事**是什么？
> 
> # 3. 演讲结构大纲（3–6 个部分）
> 按时间或逻辑顺序列出演讲结构：
> - **部分 1 标题**：一句话说明这一部分在讲什么
> - **部分 2 标题**：……
> - **部分 3 标题**：……
> （如有更多部分继续列出）
> 
> # 4. 内容还原（Detailed Reconstruction）
> 按上面的大纲，逐部分展开，还原演讲内容：
> 
> ## 4.1 部分 1 标题
> - 主要论点（1–3 条）
> - 关键解释 / 故事 / 举例
> - 如有，对应的 Slide 编号或页面提示（例如：对应 slide 3–4）
> 
> ## 4.2 部分 2 标题
> - 主要论点……
> - 关键解释……
> 
> （依次写完所有部分）
> 
> # 5. 关键数据与证据一览
> 用一个小表格列出演讲中提到的重要数据或证据（如果有）：
> 
> | 序号 | 内容摘要 | 大致来源（slide / 段落） | 你对其可信度的判断（强/中/弱） |
> |------|----------|----------------------------|--------------------------------|
> | 1    | …        | 例如：slide 5 图表       | 强                             |
> 
> # 6. 模糊点与待确认问题（可选）
> - 列出你在还原过程中发现的：逻辑跳跃、说法含糊、数据缺少来源等问题。
> ```
>
> ### 4. 风格要求
>
> - 使用中文书面语，保持简洁、清晰，但不需要逐字稿式的冗长复述；
> - 合理分段，方便后续直接复制到报告中使用；
> - 不要输出你自己的价值判断，除非在【5】和【6】中进行“可信度”和“模糊点”说明。

## 🔍 模块 2：单场 Keynote 深度洞见报告 Prompt v1.0

> 用于：在 **模块 1 还原稿已经存在** 的前提下，对“同一场 Keynote”做多维度分析。
>  使用时建议勾选：
>
> - 本场 Keynote 的还原稿（比如你保存成一个 txt/md）
> - 必要时补充 transcript / slides 作为佐证

**👉 可复制 Prompt：**

> **标题建议：**「CONF-M2 单场 Keynote 深度洞见分析」
>  **Prompt 正文：**
>
> 你是一名擅长“结构化思考 + 批判性分析”的会议研究员。
>  当前 Notebook 中已经包含了某场 Keynote 的【还原稿】，你的任务是在此基础上输出一份 **多维度洞见分析报告**。
>
> ### 1. 任务目标
>
> 在不重复大量细节叙述的前提下，从以下角度分析这场演讲：
>
> - 讲者到底想证明什么？如何证明？
> - 他/她的论点与证据是否扎实？
> - 背后的假设是什么？
> - 对我们所在行业 / 组织有什么启示与风险？
>
> ### 2. 输入假设
>
> - Notebook 里至少有一份本场演讲的【还原稿】；
> - 可能还会附带 transcript、Slides 或你的笔记；
> - 如有多份资料，请以“还原稿”为主线进行分析。
>
> ### 3. 输出格式（Markdown）
>
> ```markdown
> # 1. 核心主张地图（Main Claims Map）
> - 用 1 段话总结这场演讲的总体主张。
> - 用列表列出 2–4 个一级论点：
>   - 论点 1：……
>   - 论点 2：……
>   - 论点 3：……
> 
> # 2. 论证逻辑与结构（Reasoning & Structure）
> - 用 1–2 段话说明：讲者是如何从「现状」推导到「结论」的？  
> - 如果方便，可以用类似：
>   - 问题 → 分析框架 → 证据/案例 → 结论/建议
>   的形式来描述。
> 
> # 3. 证据与数据评估（Evidence & Data）
> - 列出 3–8 条最重要的证据/数据点，每条简要说明：
>   - 这条证据在支撑哪一个论点？
>   - 这条证据的强度：强 / 中 / 弱（基于样本、来源、可验证性等简单判断）
> 
> # 4. 隐含假设与盲点（Assumptions & Blind Spots）
> - 列出 3–5 条你从演讲中推断出来的隐含前提（例如：默认某种市场环境、监管态度不变等）。
> - 如有明显的“没讲但很关键”的维度，也写出来。
> 
> # 5. 对行业与我们的启示（Implications for Us）
> - 从“我们自己的视角”（默认你是一家在欧洲运营、重视合规与安全的科技公司）出发，提炼：
>   - 3–5 条正面启示（机会、可借鉴的做法、值得关注的趋势）
>   - 2–3 条潜在风险或需要警惕的点
> 
> # 6. 批判性评估（Critical View）
> - 用 1–2 段话评价这场演讲的：
>   - 逻辑严谨度（高/中/低，附简单理由）
>   - 数据支撑力度（强/中/弱）
>   - 原创性（高/中/低）
> 
> # 7. 后续可追问的问题（Follow-up Questions）
> - 列出 3–6 个你认为值得在后续沟通或内部讨论中继续追问的问题。
> ```
>
> ### 4. 风格要求
>
> - 用中文输出；
> - 更偏“思考备忘录”，而不是市场宣传稿；
> - 可以适度直接、尖锐，但保持理性和专业。

## 🌐 模块 3：多 Keynote 跨议题综合报告 Prompt v1.0

> 用于：**一次性勾选多场 Keynote 的还原稿 + 部分洞见报告**，输出“大会级”的综合视角。
>  例如：勾选 4–6 场相关 Session 的还原稿 + 其中 1–2 份洞见分析。

**👉 可复制 Prompt：**

> **标题建议：**「CONF-M3 多 Keynote 跨议题综合分析」
>  **Prompt 正文：**
>
> 你是一名负责大会复盘的“战略观察员”。
>  当前 Notebook 中选中的资料，包含多场 Keynote 的【还原稿】及部分洞见分析。
>  你的任务是：**跨演讲、跨议题地做综合分析，而不是简单罗列每一场的摘要。**
>
> ### 1. 任务目标
>
> - 找出这些演讲之间的 **共同主题、差异与冲突**；
> - 提炼出对行业和我们组织都重要的 **3–7 条“通用洞见”**；
> - 给出接下来在内外部沟通中应该重点继续追踪的方向。
>
> ### 2. 输出格式（Markdown）
>
> ```markdown
> # 1. 覆盖范围简述
> - 本次综合分析基于以下内容：
>   - 涉及的 Keynote / Session：列出标题 + Speaker（如果能识别）
>   - 所用资料类型：还原稿 / 洞见分析 / 少量原始转录 / Slides
> 
> # 2. 主要主题地图（Themes Map）
> - 列出 3–6 个在多场演讲中反复出现的核心主题或问题。
> - 对每个主题，用 1–2 句话说明其含义。
> 
> # 3. 各主题下的观点对比（Consensus & Divergence）
> - 按主题分组，对比不同演讲在该主题上的观点：
>   - 哪些地方形成了“共识”？（多位 Speaker 持类似看法）
>   - 哪些地方存在明显分歧或张力？
> - 如有代表性发言或数据，可简要引用。
> 
> # 4. 跨演讲的关键洞见（Cross-cutting Insights）
> - 提炼 5–7 条跨演讲的“大会级洞见”，每条包括：
>   - 洞见内容（1–2 句话）
>   - 大致由哪些演讲/Session 支撑（列出标题或 Speaker 名称）
>   - 对我们组织的潜在意义（机会 / 风险 / 提醒）
> 
> # 5. 矛盾、空白与未解问题（Tensions & Gaps）
> - 指出：
>   - 不同 Speaker 在关键议题上的明显冲突（例如：对监管走向的判断截然不同）  
>   - 大家都在回避但很关键的问题  
>   - 数据或证据明显不充分的领域
> 
> # 6. 建议的后续行动与跟进方向（Next Steps）
> - 从“内部复盘”和“外部沟通”两个角度，分别给出 3–5 条建议：
>   - 内部：例如需要开哪些内部讨论、补充哪些研究、调整哪些重点关注  
>   - 外部：例如适合在与监管对话、客户沟通中继续追问或利用的点
> ```
>
> ### 3. 风格要求
>
> - 使用中文输出；
> - 不要重写每一场的详细内容，而要**站在更高一层**看“格局与走势”；
> - 可以适度表达“优先级”（例如：哪些主题必须追，哪些可以观察）。

## 🏁 模块 4：场景化终稿 Prompt Skeleton（给你一个可填空模版）

这个模块每次都和“目标读者 + 用途”强相关，不适合写死。给你一个可复用骨架，每次只改方括号里的内容即可：

```text
你现在要扮演一名 [角色，例如：公司内部战略顾问]，目标是基于 Notebook 中选中的多份资料
（包括：若干 Keynote 还原稿、多维度洞见分析、跨议题综合报告），
为 [目标读者，例如：欧洲区高层管理团队] 生成一份用于 [用途，例如：季度战略对齐会汇报] 的总结稿。

要求：
1. 语言：使用 [中文/英文]，语气 [正式/半正式/轻松]。
2. 长度：控制在约 [X 字 / Y 页 PPT 可用文本]。
3. 结构建议：
   - 1. 背景与参会概况
   - 2. 本次大会的 3–5 个关键信息
   - 3. 与我们业务最相关的机会与风险
   - 4. 对 2025–2026 年的启示（策略/产品/合规）
   - 5. 建议的下一步行动
4. 不要逐条复述每场演讲，而是基于上游分析结果做“再综合 + 再筛选”，
   重点回答：这次会对我们“真正有用的东西是什么？”。
```
