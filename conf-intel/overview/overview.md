# NotebookLM 会议挖掘提示词库 - 架构总览

## 📖 简介

这是一个专为 NotebookLM 设计的模块化提示词库，用于从会议录音和文档中系统性地提取深度洞察。整个库采用分层工作流设计，从原始材料还原到最终成品生成，覆盖会议情报挖掘的全流程。

## 🏗️ 整体架构

### 四层工作流

```
Raw Data → M1 (还原) → M2 (洞察) → M3 (综合) → M4 (输出)
```

1. **模块 1 (M1)**: 基础还原层 - 将原始材料还原为结构化内容
2. **模块 2 (M2)**: 深度洞察层 - 多维度分析和研判
3. **模块 3 (M3)**: 综合报告层 - 跨议题整合与宏观视角
4. **模块 4 (M4)**: 终稿生成层 - 场景化输出

### 双语言支持

- **中文版** ([`cn/`](./cn/)): 直接使用中文指令，适合快速应用
- **英文版** ([`en/`](./en/)): 采用 EICO 策略（English Instructions, Chinese Output），确保逻辑精确性

## 📦 模块详细说明

### 模块 1：基础还原层

**目标**: 高保真还原会议内容，建立分析基础

**子模块**:
- **M1**: 全息会议还原 - 基础版本，使用音频转录+Slides
- **M1B**: 智能价值评估 - 增强版本，v3.2/v3.4
- **M1C**: 事实分析与系统解构 - 专业版本，v1.0

**输出**: 结构化的会议还原稿，包含核心观点、演讲结构、关键数据等

**相关文件**:
- 中文版: [`cn/conf-m1-keynote-v2.2.md`](./cn/conf-m1-keynote-v2.2.md)
- 英文版: [`en/conf-m1-keynote-en-v2.2.md`](./en/conf-m1-keynote-en-v2.2.md)

### 模块 2：深度洞察层

**目标**: 从还原稿中提取深度洞见，进行多维分析

**子模块**:
- **Universal**: 通用深度研判 - 基础分析框架
- **M2A**: 逻辑与论证审计 - 专注逻辑严密性
- **M2B**: 战略与地缘研判 - 专注战略层面
- **M2C**: 创新与知识发现 - 专注创新机会

**输出**: 包含核心主张地图、论证逻辑、证据评估、隐含假设、行业启示等

**相关文件**:
- 中文版: [`cn/conf-m2-insight-matrix-v2.2.md`](./cn/conf-m2-insight-matrix-v2.2.md)
- 英文版总览: [`en/Module 2 Deep Insight Matrix (EN Version).md`](./en/Module%202%20Deep%20Insight%20Matrix%20(EN%20Version).md)

### 模块 3：综合报告层

**目标**: 整合多场会议，提取跨议题洞见

**子模块**:
- **Executive**: 决策者摘要 (TLDR) - 精炼版本
- **Macro**: 宏观战略综合 - 全面版本

**输出**: 主题地图、观点对比、跨演讲洞见、矛盾分析、行动建议

**相关文件**:
- 英文版: [`en/conf-m3-executive-tldr-en-v2.2.md`](./en/conf-m3-executive-tldr-en-v2.2.md)

### 模块 4：终稿生成层

**目标**: 根据具体场景生成最终输出

**特点**: 提供可定制的模板框架，支持不同角色、读者和用途

**相关文件**:
- 英文版: [`en/conf-m4-draft-starter-en-v1.1.md`](./en/conf-m4-draft-starter-en-v1.1.md)

## 🎯 使用指南

### 典型工作流

1. **单场会议分析**:
   ```
   原始材料 → M1还原 → M2洞察 → M4输出
   ```

2. **多场会议综合**:
   ```
   多场材料 → 多个M1 → 多个M2 → M3综合 → M4输出
   ```

### 使用建议

1. **新手用户**:
   - 从 [`cn/`](./cn/) 中文版开始
   - 先使用 M1 还原，再尝试 M2 分析

2. **专业用户**:
   - 优先使用 [`en/`](./en/) 英文版（EICO策略）
   - 根据需求选择专业的 M2 子模块（A/B/C）

3. **大规模分析**:
   - 批量应用 M1 生成还原稿
   - 使用 M3 进行跨会议综合分析

## 💡 核心理念

1. **模块化设计**: 每个模块专注特定功能，可独立或组合使用
2. **渐进式增强**: 从基础还原到深度分析，用户可按需选择
3. **双轨并行**: 中英文版本并存，满足不同场景需求
4. **持续迭代**: 版本更新反映实际使用中的优化和改进
